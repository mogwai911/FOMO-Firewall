# prototype_sources.md — Sources（订阅源）原型规格（Web Demo / MVP）

> 更新时间：2026-02-20  
> 对齐：PRD v2 + IA v1  
> 目标：最小化 RSS 订阅管理；避免过度设计（OPML、复杂分类等）。

---

## 0) 页面定位与原则
- Sources 是配置入口，不是日常高频页。
- MVP 只支持：添加 URL、启用/禁用、删除。

---

## 1) 页面布局

### A) 顶部栏
- 标题：`订阅源`
-（可选）说明：`添加 RSS 源后，系统会自动拉取并在日报中呈现。`

### B) 添加表单（Add Source）
必须包含：
- 输入框：`RSS URL`
-（可选）名称：`显示名称`
-（可选）标签：`主题标签（逗号分隔）`
- 按钮：`添加`

校验与提示（MVP）：
- URL 非空
- 失败提示：`无法解析该 RSS，请检查链接。`

### C) Source 列表
每条 Source Item 必须包含：
- 名称（若无则用域名或 URL 简写）
- URL（灰字）
- 启用/禁用 toggle
- 删除按钮（trash）

可选：
- 最近拉取时间/状态（MVP 可不展示）

---

## 2) 关键交互
- 添加成功：列表新增一项 + toast `已添加订阅源`
- 禁用：该源不再拉取/或拉取但不出现在日报（实现二选一，MVP 建议“不拉取”）
- 删除：从列表移除（可弹确认，也可不弹；MVP 可弹一次确认）

---

## 3) 状态机
- `empty`：无订阅源（展示引导）
- `ready`：有列表
- `error`：添加失败提示

---

## 4) 必须绘制的原型帧（Codex）
- Frame S1：空态（无订阅源）+ 添加表单
- Frame S2：有列表（2–3 条 source）+ toggle + 删除
-（可选）Frame S3：添加失败提示态
