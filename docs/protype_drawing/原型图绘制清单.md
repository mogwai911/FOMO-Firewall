# FOMO Firewall 原型图绘制清单（MVP）- 给 Codex

> 目标：把 IA + PRD v2 落到可实现的交互原型（低保真即可）。  
> 约束：Web Demo；主导航 3 个入口（Digest / Sources / Memory Cards）；Evidence Pack 不做顶层入口（从 Session/卡片进入）。

---

## A. 全局框架（必须先画 1 张）
### A1) App Shell / 全局布局
- 左侧导航（3 项）：
  - Digest（日报处置）
  - Sources（订阅源）
  - Memory Cards（记忆卡）
- 顶部栏（可选）：标题 + 轻提示（toast/角标位）
- 内容区：随路由切换

---

## B. Digest（日报处置面板）— 你已保存规格，此处只列“要画哪些帧”
> Digest 的详细组件/交互按你现有文档。此处要求 Codex 至少画 3 帧。

### B1) Digest-列表折叠态（Frame 1）
必须展示：
- 列表中同时存在两种卡：
  - triage 未生成：显示 “生成建议中…”
  - triage 已生成：显示 “建议：FYI/DO/DROP”
- 每条卡都有 FYI/DO/DROP 一键按钮
- 顶部有 “待处置/已处理” tab（或最小筛选）

### B2) Digest-单条展开态（Frame 2）
必须展示（在卡片内展开或右侧抽屉均可）：
- headline（一句话结论）
- reasons ≤3
- snippets ≤2（引用块）
- next_action_hint（轻提示）
- FYI/DO/DROP 仍可点击（改判）

### B3) Digest-用户选择 DO 后（Frame 3）
必须展示：
- “你已选：DO” 的明确视觉态
- 出现 CTA：
  - “进入学习会话” 或 “继续学习”（若已有 session）
- 不跳转也能看到 CTA（可由用户点击进入）

---

## C. Sources（订阅源）— 需要画 2 帧
### C1) Sources-列表页（Frame 1）
必须包含：
- 添加 RSS 输入框（URL）
- Source 列表项：name/url（简写即可）
- 每项操作：启用/禁用 toggle + 删除按钮
- 空状态：无 source 时，提示 “添加 RSS 源开始使用”

### C2) Sources-添加成功后的状态（Frame 2）
必须包含：
- 列表新增一项
- 显示 enabled 状态
-（可选）toast：已添加

---

## D. Session（学习会话）— 需要画 4 帧（这是 MVP 第二关键页）
> 核心原则：退出=暂存；默认保存全量对话；闪卡/证据包均为异步 Job，不阻塞。

### D1) Session-初始进入（Frame 1）
必须包含：
- 顶部线索信息：标题/来源/原文链接
- 问题卡区域：
  - 显示 3–5 个问题卡（可点击发送）
  - 也允许用户直接在输入框提问
- 对话区为空/欢迎语
- 右侧或底部有 “生成区”（按钮+状态位）：
  - 生成闪卡（后台）
  - 生成证据包（后台）
  - Job 状态占位（queued/running/done/failed）

### D2) Session-对话进行中（Frame 2）
必须包含：
- 至少 2 轮对话消息（user/assistant）
- 输入框与发送按钮
- 仍可看到 “生成区”（按钮可点）

### D3) Session-点击生成后（异步）状态（Frame 3）
必须包含：
- 用户点击 “生成闪卡（后台）”
- UI 显示 Job 状态变化：
  - queued 或 running
- 用户仍然可以继续聊天（不弹阻塞）

### D4) Session-退出会话（Frame 4）
必须包含：
- “退出/返回日报”按钮
- 退出后行为说明（轻提示即可）：会话已保存，可随时继续
- 返回 Digest 后，该 signal 卡出现 “继续学习”入口（可在 Digest Frame 3 复用体现）

---

## E. Memory Cards（记忆卡）— 需要画 3 帧（高频结果页）
> 核心：像背单词一样划卡；能回溯 Session / Evidence。

### E1) Memory Cards-划卡复习模式（Frame 1）
必须包含：
- 当前卡片正面：Question 或要点标题
- 翻面动作：显示 Answer
- 操作：下一张 / 标记熟悉（按钮即可，细节可简化）
-（可选）卡片来源/标签（轻信息）

### E2) Memory Cards-卡片库列表（Frame 2）
必须包含：
- 搜索框（关键词）
- 列表项显示：question 摘要 + 来源
- 点击进入卡片详情（抽屉或页面）

### E3) Card Detail-回溯入口（Frame 3）
必须包含：
- 完整 Q/A
- 两个跳转按钮：
  - “回溯会话（Session）”
  - “查看证据包（Evidence Pack）”（若未生成则提示去生成/生成中）

---

## F. Evidence Pack（证据包）— 可选但建议画 2 帧（低频）
> MVP 可不放顶层导航，仅从 Session 或卡片进入。

### F1) Evidence Pack-详情页（Frame 1）
必须包含：
- summary（可读摘要）
- key_quotes（1–3 条引用块）
- links（1–3 条链接）
- “展开全量对话”（折叠区占位即可）

### F2) Evidence Pack-展开全量对话（Frame 2）
必须包含：
- 折叠区展开后展示对话片段（来自 SessionMessage）
- “继续聊”（跳转回 Session）

---

## G. 系统态/空态（必须至少各画 1 个）
### G1) Digest 空态
- 无线索：提示 “今天没有新的线索”
### G2) triage 失败态（在 Digest 卡片上体现即可）
- 显示 “建议生成失败”，但 FYI/DO/DROP 仍可手动处置
### G3) Job 失败态（在 Session 的生成区体现即可）
- 显示 failed + “重试”按钮（可选）

---

## H. 最小交付要求（Codex 输出的原型必须满足）
- 至少包含：Digest（3帧）+ Sources（2帧）+ Session（4帧）+ Memory Cards（3帧）
- 所有关键门（入口/出口）可见：
  - Digest → Session（DO）
  - Session → Digest（退出=暂存）
  - Session 触发异步生成（不阻塞）
  - Memory Cards → Session / Evidence Pack（回溯）
